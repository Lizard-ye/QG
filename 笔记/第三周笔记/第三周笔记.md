## 静态数码管原理（段选、位选）
* * *
### 数码管
#### 1.结构
数码管（也称为七段显示器）由 7个发光二极管（LED） 组成，每个二极管称为一个 段。
这7个段分别命名为 a, b, c, d, e, f, g，还有一个可选的小数点段 dp。
![7debc11e5790a9319b5eab69e1f8b46c.png](../_resources/7debc11e5790a9319b5eab69e1f8b46c.png)

**简单来说，静态数码管是一种显示方式，它让数码管上的每一个发光段（LED）都拥有自己独立的控制线。**



#### 工作方式：
当你想显示一个数字（比如“8”）时，单片机会一次性把控制信号（高电平或低电平）发送给对应的引脚，点亮对应的段。只要单片机不改变这些引脚的电平，数码管就会一直显示这个数字，直到你让它熄灭或改变。**分为：**
1. 共阴极数码管
共阴极：所有段的阴极连接在一起，通常接地（GND）。
显示逻辑：要显示某个段，需要将该段的阳极接高电平（VCC）。
2. 共阳极数码管
共阳极：所有段的阳极连接在一起，通常接电源（VCC）。
显示逻辑：要显示某个段，需要将该段的阴极接低电平（GND）。

| **类型**   | **公共端连接** | **点亮条件**       | **常用驱动方式**     |
|------------|----------------|--------------------|----------------------|
| **共阴极** | COM端接地(GND) | **段接高电平**点亮 | 单片机直接驱动       |
| **共阳极** | COM端接电源(VCC)| **段接低电平**点亮 | 需三极管/驱动芯片    |

**注**：51单片机常用**共阴极数码管**（IO输出高电平点亮段）

#### 优点：
显示稳定，无闪烁，程序控制简单。

#### 缺点：
占用单片机的I/O口（引脚）资源非常多。一个数码管通常有8个段（7个笔段+1个小数点），如果直接驱动，一个数码管就要占用8个引脚。如果系统有4个数码管，就需要占用32个引脚，这对于引脚数量有限的单片机来说是非常“奢侈”的。
* * *

### 控制基础 

#### 段选和位选：
#### 1. 段选（Segment Selection）
- **作用**：控制**显示什么字符**（如数字0-9）
- **实现**：通过8个IO口（P0/P1/P2）输出**段码值**（编码表决定）
- **连接**：单片机IO → 数码管段引脚(a,b,c,d,e,f,g,dp)

#### 2. 位选（Digit Selection）
- **作用**：控制**哪个数码管**亮（适用于多位数码管系统）
- **实现**：通过额外IO口输出**位码值**（1表示选中，0关闭）
- **连接**：单片机IO → 数码管公共端(COM1, COM2...)

## 矩阵键盘原理（行扫描 / 列扫描），按键识别算法

### 51单片机的矩阵键盘原理及按键识别算法

矩阵键盘是一种常用的输入设备，可以通过较少的I/O口线实现较多按键的扫描

### **一、矩阵键盘原理**
![66227d1f4df3892cccf463d4e4ce598e.png](../_resources/66227d1f4df3892cccf463d4e4ce598e.png)

#### 1. 硬件结构
- **4x4矩阵键盘**：由4行和4列组成，共16个按键。
- **连接方式**：
  - 行线（Row）：通常连接到单片机的P1.0~P1.3。
  - 列线（Column）：通常连接到单片机的P1.4~P1.7。
  - 每个按键位于行线和列线的交叉点上。

**结构图**
  ![f001b0fe674742024bc9f2542ca059dd.png](../_resources/f001b0fe674742024bc9f2542ca059dd.png)

#### 2. 工作原理
- **初始状态**：所有行线输出高电平（1），所有列线配置为输入模式。
- **按键按下**：当某个按键被按下时，对应的行线和列线之间形成通路，使得列线上的电平发生变化。
- **扫描检测**：通过逐行或逐列扫描，检测哪一行和哪一列的电平发生变化，从而确定哪个按键被按下。

#### 扫描？
![33cfcaee1d6d46c9615eea2dae99e4df.png](../_resources/33cfcaee1d6d46c9615eea2dae99e4df.png)
* * *

### **二、行扫描与列扫描**

#### 1. 行扫描
- **步骤**：
  1. 将所有行线设置为高电平（1）。
  2. 将所有列线设置为输入模式，并读取列线的电平。
  3. 逐行将行线设置为低电平（0），并读取列线的电平。
  4. 如果某列线的电平从高变低，说明该行和该列的按键被按下。

##### （逐行扫描同一时间只给其中一行0，其他给1，然后看列）
——若蜂鸣器会响——p15口占用问题——普中开发板无影响--p25口接beep

#### 2. 列扫描
- **步骤**：
  1. 将所有列线设置为低电平（0）。
  2. 将所有行线设置为输入模式，并读取行线的电平。
  3. 逐列将列线设置为高电平（1），并读取行线的电平。
  4. 如果某行线的电平从高变低，说明该行和该列的按键被按下。

***

### **三、按键识别算法**

#### 1. 基本思路
- **去抖动**：按键按下时会有机械抖动，需要通过延时来消除抖动。
- **按键识别**：通过行扫描或列扫描检测按键位置。
- **按键值映射**：将检测到的按键位置映射为具体的键值。

### 联动数码管显示

**要点**

- 需要将数码管键值分解为十位个位

***拆解键值（把号码拆成单个数字）***

***比如按键是 15 号：
十位：15 ÷ 10 = 1（取整）
个位：15 ÷ 10 余数 = 5***

```
#include <REGX52.H>
#include "Delay.h"
#include "lcd1602.h"
#include "matrixKey.h"
#include "nixie.h"


unsigned char KeyNum;
unsigned char last_key = 0;
unsigned char shiwei=0;
unsigned char gewei=0;


void main()
{


	while(1)
	{
		KeyNum=matrixKey();
    if(KeyNum!=0)
    {
			last_key=KeyNum; // 记住按键值
    }

		shiwei=last_key/10;//分解求十位
		gewei=last_key%10;//取余求个位
//分别显示
		nixie(1,shiwei);
		nixie(2,gewei);
		
		Delay(3);
		
	}

}

## 矩阵键盘与数码管联动/密码锁
基于lcd1602实现
```
#include <REGX52.H>
#include "Delay.h"
#include "lcd1602.h"
#include "matrixKey.h"

unsigned char KeyNum=0;
unsigned int Password=0,Count;

void main()
{
	LCD_Init();
	LCD_ShowString(1,1,"password: ");
	while(1)
	{
		KeyNum=matrixKey();
		if(KeyNum)
		{
			
			if(KeyNum<=10)//若s1-s10按下，输入密码
			{
				
				if(Count<4)//密码4位，按下次数小于四才计入（只计入四位）
				{
					Password*=10;//密码左移一位
					Password+=KeyNum%10;//获取一位密码
					Count++;//计次加1
				}
				LCD_ShowNum(2,1,Password,4);//更新显示
			}
			if(KeyNum==11)//若s11按下（确认）
			{
				if(Password==4321)//密码正确时
				{
					LCD_Init();
					LCD_ShowString(1,1,"Hi,Lizard");
					Password=0;//密码清零
					Count=0;//计次清零
				}
				else//密码错误时
				{
					LCD_Init();
					LCD_ShowString(1,1,"Error");
					LCD_ShowString(1,8,"Try again");
					Password=0;//密码清零
					Count=0;//计次清零
				}
			}
			if(KeyNum==12)
			{
				LCD_Init();
				Password=0;//密码清零
				Count=0;//计次清零
				LCD_ShowString(1,1,"password: ");
			}

		}
	}
}
